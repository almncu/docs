# drop_dtmf 

由 Ryan Harris 创建，由 Attila Gulyas 在2019年12月23日进行了最后修改

## drop_dtmf

boolean 在应答之前设置此项来处理入站通道，或在桥接/发起呼叫之前设置此项来处理出站通道，以防止将DTMF事件发送到通道。

仅在RFC2833中进行了测试，可能也适用于INFO / inband。 请参阅[FS-4769](https://freeswitch.org/jira/browse/FS-4769?src=confmacro) -[Authenticate](https://freeswitch.org/confluence/plugins/servlet/applinks/oauth/login-dance/authorize?applicationLinkID=c4b15fa3-ad57-3c05-af05-8c167cb92266)以查看问题详细信息。 提交[60f7849cbe72](https://freeswitch.org/stash/projects/FS/repos/freeswitch/commits/60f7849cbe72)。

TODO [RFC 2833](https://tools.ietf.org/html/rfc2833)被[RFC 4733](https://tools.ietf.org/html/rfc4733)废弃。

**用法**

```xml
<action application="set" data="drop_dtmf=true"/>
<action application="answer"/>
```

```xml
<action application="export" data="nolocal:drop_dtmf=true"/>
<action application="bridge" data="sofia/internal/100@1.2.3.4"/>
```

```xml
<action application="bridge" data="{drop_dtmf=true}sofia/internal/100@1.2.3.4"/>
```
