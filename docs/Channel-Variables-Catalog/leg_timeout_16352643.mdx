## leg_timeout

整数，指定呼叫字符串中每个 `leg` 的超时时间。可以在每个 `[]` 中使用，但不能在呼叫字符串的全局 `{}` 中使用。对于全局设置，请使用 [originate_timeout](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#originate_timeout)。

您还可以使用 [leg_progress_timeout](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#leg_progress_timeout) 来指定我们在获取媒体之前等待的最长时间（无论是早期媒体、振铃还是应答），从而避免针对特定线路转到语音邮箱。

如果您使用了 `group confirm`，则可以通过使用 [group_confirm_cancel_timeout](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#group_confirm_cancel_timeout) 通道变量来取消超时。如果还使用了 [leg_delay_start](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#leg_delay_start)，则 `leg_timeout` 只有在分机开始连接后才会开始超时计数。

**用法**

```xml
<action application="bridge" data="[leg_timeout=15]user/hastoanswerquickly/some.domain.com,[leg_timeout=60]user/hasaminutetoanswer@some.domain.com"/>
```
