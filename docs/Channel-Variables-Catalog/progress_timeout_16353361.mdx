# 进度超时

由Ryan Harris创建，上次修改于2018.02.07

## 进度超时

整数。这是我们在等待获取媒体（无论是早期媒体、振铃还是应答）之前的最长时间，例如在放弃路由之前的最大呼叫处理延时（PDD） - 即从发送INVITE到接收到第一个振铃响应，例如：SIP/2.0 180 Ringing或183。典型的固定电话PDD在3秒以下，移动电话在8秒以下。

> 我将我的进度超时设置为13秒，以适应慢速移动连接，但是它可能可以更短。 - [Avi Marcus](https://freeswitch.org/confluence/display/~avimarcus)

两种情况：

1. 如果运营商在其平台上遇到问题，**所有**呼叫都可能会超时。
2. 如果这不是运营商的问题 - 如果单个运营商在13秒内超时，并且用户实际上等待了，那么这次呼叫很少连接到其他运营商（尽管它们都在13秒后过期）。

另请参阅：[早期媒体](../FreeSWITCH-Explained/Codecs-and-Media/Early-Media/index.mdx#about)

**用法**

```xml
<action application="set" data="progress_timeout=20"/>
```
