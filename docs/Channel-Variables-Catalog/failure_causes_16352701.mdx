## 失败原因

string控制将哪些失败原因视为对桥梁的失败。除非将`continue_on_fail`设置为`true`，否则这将更改[continue_on_fail](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#continue_on_fail)的默认失败值。

**用法**

```xml
<action application="set" data="failure_causes=USER_BUSY,NO_ANSWER"/>
```

使用Q.850故障原因代码：

```xml
<action application="set" data="failure_causes=487"/>
```
