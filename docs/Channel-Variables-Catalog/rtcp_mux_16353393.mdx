# rtcp_mux

创建者：Ryan Harris，最后修改日期：2018.02.07

## rtcp_mux

布尔类型的通道变量，用于强制在发送的INVITE请求中使用rtcp-mux。

## 评论:

| 我在呼叫的B端将rtcp_mux设置为false。从信令的角度来看，发送给服务提供商的邀请请求是正确的，即在offer/INVITE中显示了分离的rtp和rtcp端口。然而，freeswitch在向服务提供商发送媒体时并没有将实际的RTP和RTCP流分开。服务提供商发回的200 OK消息中没有进行rtcp多路复用，而服务提供商期望freeswitch将RTP和RTCP流分开。然而，freeswitch将RTP和RTCP流都发送到同一个端口。有关如何解决此问题的建议吗？ ![](/images/icons/contenttypes/comment_16.png) 发表者：hailegr，发表于2020年2月25日19:33 |
