# local_ip_v6

由Duvid Rottenberg创建，于2018.04.24上次修改

## local_ip_v6

字符串

这个全局变量是由FreeSWITCH核心动态设置的，它通过向底层操作系统询问运行FreeSWITCH的服务器的IPv6地址（如果支持IPv6）来获取。如果服务器有多个IPv6地址，那么该变量将被设置为操作系统返回的第一个地址。在[默认配置](../FreeSWITCH-Explained/Configuration/Default-Configuration_6587388.mdx#mod_vp8)中，该变量用于设置IPv6 SIP和Verto配置文件。

你不能覆盖此全局变量的值，因为FreeSWITCH核心会每分钟重置它。如果您的服务器有多个IPv6地址，并且您想要强制FreeSWITCH使用特定的IPv6地址，您可以创建一个名为force\_local\_ip\_v6的全局变量，并将其设置为您首选的IP地址。您还必须将local\_ip\_v6变量设置为该IP地址，只需将local\_ip\_v6设置为force\_local\_ip\_v6的值即可。

**示例：force\_local\_ip\_v6**

```xml
<X-PRE-PROCESS cmd="set" data="force_local_ip_v4=1.1.1.1"/>
<X-PRE-PROCESS cmd="set" data="local_ip_v4=$${force_local_ip_v4 }"/>
```

FreeSWITCH不会验证您为`force_local_ip_v6`输入的IPv6地址是否实际上指向服务器，您需要确保使用正确路由到服务器的有效IP地址。
