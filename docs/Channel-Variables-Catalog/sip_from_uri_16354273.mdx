## sip_from_uri

[string] 发送INVITE请求的端点的SIP URI。

**用法**

```xml
<action application="export" data="sip_from_uri=${sip_from_uri}"/>
```

一些SIP提供商坚持以非正常方式处理呼叫者ID。在大多数情况下，您可以使用[effective\_caller\_id\_name](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#effective_caller_id_name)和[effective\_caller\_id\_number](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#effective_caller_id_number)来设置呼叫者ID的出站（B leg）呼叫。但在某些情况下，SIP提供商可能不喜欢这样做，可能只显示“匿名”或其他默认的呼叫者ID信息。

上述示例获取传入SIP呼叫的From URI，并将该值导出到B leg。当存在以下场景时，这非常方便：

```xml
Alice ==> FreeSWITCH桥接 ==> Bob
```

您希望将Alice的呼叫者ID信息发送给Bob，而不是FreeSWITCH的呼叫者ID信息。通常这是"自动工作"的，但如果不行，您可以尝试像上面一样导出sip\_from\_uri。
