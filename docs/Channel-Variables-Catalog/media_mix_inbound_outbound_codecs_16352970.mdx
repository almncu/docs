创建者：Ryan Harris，最后修改日期：2018年02月07日

## 媒体混合入站/出站编解码器

布尔值，设置将编解码器列表提供给B端。

FreeSWITCH以前会从入站A端获取编解码器，并将它们与配置的出站编解码器列表混合提供给B端。使用大量编解码器时，可能会超过UDP消息的最大传输单元（MTU），导致呼叫建立失败。

自[FS-8321](https://freeswitch.org/jira/browse/FS-8321?src=confmacro)起，为了查看详细问题说明，您需要-[验证](https://freeswitch.org/confluence/plugins/servlet/applinks/oauth/login-dance/authorize?applicationLinkID=c4b15fa3-ad57-3c05-af05-8c167cb92266) ，旧的行为现在需要将变量`media_mix_inbound_outbound_codecs=true`设置为默认行为，即在出站B端提供与入站A端提供的完全相同的编解码器。

**用法**

```xml
<action application="set" data="media_mix_inbound_outbound_codecs=true"/>
```
