# rtp_negotiate_near_match 

由Ryan Harris创建，上次修改于2018.02.07

## rtp_negotiate_near_match

boolean 将这个设置为true将会导致FreeSWITCH在入站编解码协商时，也考虑编解码器PTIME。如果您收到一个以PCMU@30i呼叫且列表上只有PCMU@20i，它将在默认情况下选择与编解码器和ptime匹配的任何其他选项。这将改变行为并匹配PCMU@30i，尽管ptime不匹配。当然，这只在端点列表上存在其他匹配编解码器的情况下有效。

**用法**

```xml
<action application="set" data="rtp_negotiate_near_match=true"/>
```
