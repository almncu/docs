## group_confirm_key

在 `group_confirm_key` 变量中，您定义了被呼叫方应发送给 FS 的 DTMF，以便将呼叫桥接。如果发送的 DTMF 错误或未发送任何 DTMF，则不会桥接呼叫，并且将重复播放 WAV 文件。

**用法**

```xml
<action application="set" data="group_confirm_key=1" />
```
