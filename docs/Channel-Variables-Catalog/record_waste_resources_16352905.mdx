## record_waste_resources

默认情况下，录音不会发送RTP数据包。这通常是可接受的，但对于较长的录音或根据网关的RTP计时器而定，您的通道可能会因为`MEDIA_TIMEOUT`而挂断。设置此变量将“浪费”带宽，即使在录音期间也会发送RTP。该值可以是`true`，`false`或`[期望的静音因子]`。默认情况下，如果您设置`record_waste_resources=true`，静音因子是1400。

**用法**

```xml
<action application="set" data="record_waste_resources=true"/>
```

```xml
<action application="set" data="record_waste_resources=1300"/>
```
