# sip_refer_continue_after_reply 

由Bao Nguyen创建，最后修改于2021年11月02日

## sip_refer_continue_after_reply

将`sip_refer_continue_after_reply`通道（或全局）变量设置为`true`以启用，在使用deflect()或uuid_deflect()发送REFER之前。默认情况下，FreeSWITCH在成功发送REFER后会结束通话（在随后的NOTIFY消息的正文中带有200 OK）。

启用后，如果不使用代理参考(proxy-refer)，FreeSWITCH将处理REFER并在`sip_refer_target_status_code`中设置最终结果，在`sip_refer_target_provisional_status_code`中设置临时状态。通话将一直保持直到显式地结束。

自FreeSWITCH v1.10.7版本发布以来可用。

在测试rfc4579 REFER方法以请求将新资源添加到会议中时，这很有用。

## 用法

```xml
<action application="set" data="sip_refer_continue_after_reply=true"/>
```

## 示例

```xml
<action application="set" data="sip_refer_continue_after_reply=true"/>
<action application="deflect" data="sip:someone@somewhere.com"/>
<action application="echo" data="" />
```
