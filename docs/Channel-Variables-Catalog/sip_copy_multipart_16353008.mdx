## sip_copy_multipart

boolean类型FreeSWITCH仅支持带有多部分消息主体的`INVITE`。FreeSWITCH不支持带有多部分消息主体的响应消息，如`183 Session Progress`或`200 Ok`。通常，SIP消息主体只有一个使用`application/sdp` MIME类型的MIME部分中的SDP。SIP规范允许使用`multipart/mixed` MIME类型定义多个消息主体。在这种情况下，FreeSWITCH将尽力找到带有SDP的MIME部分，并像通常一样解析它。然而，您可以通过该变量改变FreeSWITCH处理多部分消息主体并使用桥接的行为。

![(info)](/images/icons/emoticons/information.svg) FreeSWITCH将会“正确处理”，并像往常一样在B leg上附上由FreeSWITCH生成的应用层/SDP类型（根据您的设置）。其他非SDP的MIME部分将会直接传递。

**用法**

要在使用桥接时让FreeSWITCH保留多个MIME部分（默认行为）：

```xml
<action application="set" data="sip_copy_multipart=true"/>
```

要在使用桥接时让FreeSWITCH剥离多个MIME部分：

```xml
<action application="set" data="sip_copy_multipart=false"/>
```
