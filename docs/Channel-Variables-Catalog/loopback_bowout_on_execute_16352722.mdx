# loopback_bowout_on_execute

由 Ryan Harris 创建，最后修改于2018年02月10日

## loopback_bowout_on_execute

将此参数设置为`true`，可以使单腿[回环通道](../FreeSWITCH-Explained/Bit-Bucket/For-Consideration/mod_bridgecall_13173999.mdx#loopback)“退出”（删除回环腿）呼叫。

`loopback_bowout_on_execute`仅适用于单腿呼叫。您不必设置它：`loopback_bowout`默认为`true`。它尝试通过执行[uuid\_bridge](../FreeSWITCH-Explained/Modules/mod_commands_1966741.mdx#uuid_bridge)来移除回环。

当设置`loopback_bowout=false`时，桥接的回环呼叫将产生4个腿（a-leg、loopback-a、loopback-b、b-leg）。

当设置`loopback_bowout=true`时，桥接的回环呼叫将产生2个腿（a-leg、b-leg）。

然而，在呼叫建立阶段，即使使用`loopback_bowout=true`，始终会存在4个腿。只有在成功桥接a-leg和b-leg后，loopback-a和loopback-b才会被销毁。

**用法**

```xml
<action application="set" data="loopback_bowout_on_execute=true"/>
```
