# continue_on_fail

由Ryan Harris创建，上次修改日期为2018年2月7日

## continue_on_fail

字符串，控制在无法联系上被呼叫方（忙碌/离线）时的处理方式。如果为`true`，则继续处理呼叫计划。如果为`false`，则停止处理呼叫计划。还可以包含在失败时继续执行的返回消息。

**用法**

```xml
<action application="set" data="continue_on_fail=true"/>
```

```xml
<action application="set" data="continue_on_fail=NORMAL_TEMPORARY_FAILURE,USER_BUSY,NO_ANSWER,NO_ROUTE_DESTINATION"/>
```

使用Q.850原因代码：

```xml
<action application="set" data="continue_on_fail=3,17,18,27"/>
```
