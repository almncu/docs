## passthru_ptime_mismatch

布尔值，如果来自 leg A 和 leg B 的 `ptime` 不匹配，并且使用了 [mod\_com\_g729](../FreeSWITCH-Explained/Modules/mod_com_g729_1049284.mdx#about)，通话通常会使用编解码器重新打包 RTP 流。

使用此参数，[mod\_com\_g729](../FreeSWITCH-Explained/Modules/mod_com_g729_1049284.mdx#about) 将在不解码/编码的情况下重新打包，就像 [mod\_g729](../FreeSWITCH-Explained/Modules/mod_g729_3375549.mdx#about) 一样。

**使用方法**

在桥接之前，需要将其设置为`{}`。如果在桥接之前使用`export`进行设置，可能无效。

```xml
<action application="bridge" data="{passthru_ptime_mismatch=true}sofia/gateway/trunk/$1"/>
```

全局设置在 `vars.xml` 中

```xml
<X-PRE-PROCESS cmd="set" data="passthru_ptime_mismatch=true"/>
```
