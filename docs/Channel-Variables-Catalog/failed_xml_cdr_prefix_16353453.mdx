## failed_xml_cdr_prefix

如果您在A路设置了这个，而且所有的B路呼叫都失败了，那么将会生成一个完整的XML CDR报告，并将其设置为一个变量，这个包括在分叉拨号期间。

假设您尝试呼叫`sofia/profile/a@xxxxxxx,sofia/profile/b@xxxxxxx`。

然后它完全失败，在进行呼叫之前，您设置`failed_xml_cdr_prefix`为"bad\_call"。

接着您将得到`${bad_call_1}`和`${bad_call_2}`，它们分别是一个包含所有变量等的完整XML报告。

**用法**

```xml
<action application="set" data="failed_xml_cdr_prefix=failinggw" />
```
