# bridge_hangup_cause 

 由Ryan Harris创建，最后修改日期为2018.02.07

## bridge_hangup_cause

string 设置为呼叫的最后一个连接的B端挂机原因。如果您设置了`continue_on_fail=true`和`hangup_after_bridge=false`，您可以检查这个值，以了解呼叫实际上发生了什么情况。例如，您可以在桥接之后执行`execute_extension`应用程序，然后在`${bridge_hangup_cause}`上进行条件检查，看看它是否包含`MEDIA_TIMEOUT`，然后重新拨号或将呼叫转接到手机。有关挂机原因的列表，请参阅[挂机原因](../FreeSWITCH-Explained/Troubleshooting-Debugging/Hangup-Cause-Code-Table_3964945.mdx#about).

**用法**

```xml
<action application="log" data="1 B-leg挂机原因: ${bridge_hangup_cause}"/>
```
