## bridge_early_media

默认情况下，这个值为false。设置为true，则桥接使用b-leg的实时音频作为a-leg的拨铃音。设置`bridge_early_media=true`意味着早期媒体将被缓冲。

**如果满足以下条件，请考虑将其设置为true：**

* 您正在使用[回环信道](../FreeSWITCH-Explained/Bit-Bucket/For-Consideration/mod_bridgecall_13173999.mdx#loopback)执行桥接到一个发送早期媒体并且接收到的早期媒体音频质量降低的端点。

注意：设置`bridge_early_media=true`会导致比`bridge_early_media=false`更高的资源消耗，但可以改善早期媒体的音质。在桥接之前或桥接的拨号字符串中设置`bridge_early_media`。

* 您在originate或bridge中使用多个目标（例如：_originate{globalVar=global}\[channelVar=channel1\]sofia/gateway/GW1/123456|\[channelVar=channel2\]sofia/gateway/GW2/123456)_，其中您需要接收网络生成的呼叫进展音（例如：铃声和网络通告）。您可以与continue\_on\_fail结合使用，这样您可以仅对特定释放原因进行调优的备用方案。
