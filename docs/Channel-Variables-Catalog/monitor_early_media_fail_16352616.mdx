# monitor_early_media_fail

由Ryan Harris创建，上次修改日期为2018.02.07

## monitor_early_media_fail

字符串，用于监视早期媒体的故障条件，例如忙音。这样可以在忽略早期媒体时更快地处理失败的呼叫。

语法是以以下格式用 `!` 分隔的一系列早期媒体条件：

`condition_name:number_of_hits:tone_detect_frequencies`

| condition\_name           | 用户自定义错误条件的名称                          |
| ------------------------- | ---------------------------------------- |
| number\_of\_hits          | 必须听到音调的次数，才会被视为失败                  |
| tone\_detect\_frequencies | 要侦听的频率（用 + 而不是 , 分隔）。请参阅 [tone\_detect](../FreeSWITCH-Explained/Modules/mod-dptools/6586957.mdx#about) |

![(warning)](/images/icons/emoticons/warning.svg)只有在设置了 `ignore_early_media` 为 true 时，此变量才起作用。

**用法**

```xml
<action application="bridge" data="{ignore_early_media=true,monitor_early_media_fail=user_busy:2:480+620!destination_out_of_order:2:1776.7}sofia/dial/string"/>
```
