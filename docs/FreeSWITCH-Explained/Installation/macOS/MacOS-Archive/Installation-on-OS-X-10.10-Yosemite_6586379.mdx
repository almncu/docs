### 创建 /usr/local 子目录

在 macOS 10.10 Yosemite 中，默认没有提供 /usr/local 目录，所以需要在终端中运行以下命令来创建：

```xml
  cd /usr
  sudo mkdir local                          # 创建 /usr/local 目录
```

### 下载 Xcode（自2017年12月起不再需要）

下载 Xcode 需要一些时间，它将被放置在 Applications 文件夹（/Applications 目录）中，**命令行工具（Command Line Tools）**位于 /Applications/Xcode.app/Contents/Developer 目录中。

此 Xcode 下载需要您使用您的 Apple ID 登录并同意软件协议：  
Yosemite 10.10：从 [Xcode 7.2.1 下载](https://developer.apple.com/services-account/download?path=/Developer%5FTools/Xcode%5F7.2.1/Xcode%5F7.2.1.dmg) 下载并安装 Xcode 7.2.1，然后启动 Xcode。按照 [重命名 Xcode 目录（截止到 Xcode 7.1）](Installation-on-OS-X-10.10-Yosemite_6586379.mdx#rename-xcode-directory-as-of-xcode-71) 进行操作。

## OS X 开发者工具

OS X 开发者工具包括两个部分：用于开发和测试 OS X 图形界面应用程序的 [Xcode 7](http://developer.apple.com/technologies/tools/xcode.html) OS X 软件开发工具包，以及提供生成非图形界面应用程序（如 FreeSWITCH™）和其先决条件所需的 [LLVM](http://en.wikipedia.org/wiki/LLVM)、[Clang](http://en.wikipedia.org/wiki/Clang)、[Git](http://en.wikipedia.org/wiki/Git%5F%28software%29) 等工具的**命令行工具**。命令行工具已包含在 Xcode 中并自动安装。Apple 开发者网站提供了**独立的 CLT**，但_**无法使用，因为 Homebrew 在没有** **Xcode/CLT 版本的情况下无法安装。**_

### 安装 Xcode 7.2

Xcode 7.2在[OS X App Store](https://itunes.apple.com/us/app/xcode/id497799835?mt=12&ign-mpt=uo%3D4)中已经不再提供，因此需要从[OS X Developer网站](http://developer.apple.com/devcenter/mac/index.action)下载，这需要至少一个免费会员资格。下载Xcode需要一些时间，并将其放置在Applications文件夹(/Applications目录)中，CLT位于/Applications/Xcode.app/Contents/Developer。_**安装完成后，请启动Xcode以接受许可协议。**_

### 重命名Xcode目录（从Xcode 7.1开始）

Xcode 7安装的命令行工具适用于OS X 10.11，这些工具在OS X 10.10上运行良好。然而，Homebrew在OS X 10.10下运行时会检查10.10版本。如果没有找到它们，它会提示你单独安装它们。如果你将命令行工具目录重命名（别名不起作用）如下所示，这是不必要的：

1. 打开Applications文件夹，选择Xcode。
2. 按住Control键单击Xcode，以显示上下文菜单，然后选择Show Package Contents。
3. 导航到：Contents –> Developer –> Platforms –> MacOSX.platform –> Developer –> SDKs –> MacOSX10.11.sdk
4. 选中MacOSX10.11.sdk，按下Enter键
5. 将10.11更改为10.10，然后按下Enter键。目录名称必须为MacOSX10.10.sdk
