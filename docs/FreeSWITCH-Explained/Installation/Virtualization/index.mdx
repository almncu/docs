# 虚拟化

## 关于

虚拟化服务器因节省物理空间和电力而越来越受欢迎。然而：

许多FreeSWITCH安装已成功部署在虚拟环境中。由于虚拟主机在性能和提供的服务方面差异很大，而且您的应用程序将具有自己的特定需求，您必须测试特定的虚拟主机与FreeSWITCH应用程序的组合。例如，一个大型视频会议服务器与一个语音中转开关有不同的需求。请向FreeSWITCH社区寻求指导和经验，以助您做出选择。

在虚拟机上成功的关键是确保来自硬件的计时源与虚拟机共享。一些虚拟机，如vmware，会进行虚拟计时，这是不够好的。

## OpenVZ

为了加快您构建新的FreeSWITCH安装的速度，FS开发团队准备了一个包含所有必要库和依赖关系的OpenVZ容器:

<http://files.freeswitch.org/openvz/>

默认的root密码是'freeswitch'，请在继续之前将其更改。

然后按照[标准的FreeSWITCH安装步骤](../index.mdx#about)或者按照登录到新容器时显示的motd指令进行操作，很快就能上手运行。

如果您的FS运行的流量很大，请使用以下位置的默认模板：

/etc/vz/conf/ve-unlimited.conf-sample

仅在<https://jira.freeswitch.org/browse/FSVIRT>上报告关于容器的问题。
