# 使用邮件列表

## 关于

邮件列表是寻求有关FreeSWITCH相关问题/问题的主要来源。然而，许多用户无法充分利用该列表，因为他们在提出问题和与列表成员交互方面缺乏经验。本页面旨在解决这个问题。包括核心开发人员在内的许多列表成员都花费了大量的时间和精力来耐心回答问题，我们用户应该花费同样的努力来正确提出问题。请尝试按照下面概述的步骤，使邮件列表上的体验更愉快和有益。

### 做好功课！

* 大部分时候，Google可以回答你的问题。
* 不要期望能够立即获得一个‘一次性/复制粘贴’的答案，可能需要进行一些工作。
* 不要在第一页放弃，通常需要查看400-600个结果（取决于每个结果的相关性）。
* 不要害怕深入研究FreeSWITCH的源代码（这是学习和找到导致问题的提示的好方法）。
* 浏览freeswitch-users的邮件列表存档。

### 如何使用Google

假设你正在寻找与"sofia.conf"相关的信息，仅仅输入这个关键词到Google中通常是不够的，你需要提供一些上下文。以下是一些示例：

```text
"sofia.conf" freeswitch
"sofia.conf.xml" freeswitch
"mod_sofia" freeswitch (config|configuration)
"sofia.c" freeswitch drops
freeswitch "call (drops|dropped)"
freeswitch "mod_xml_curl" debug
"fscli" freeswitch
"MySQL" freeswitch (CDR|cdrs)
```

### 小窍门

尽可能显示更多的调试信息；

```text
console loglevel debug
```

获取SIP跟踪；

```text
sofia global siptrace on
sofia global siptrace off
```

### 准备你的问题

#### 要做的事情

* 使用一个有意义的主题行 - “新手需要帮助” 可能会给你带来捐赠，但其它情况下是没有用的。使用你在搜索中使用的关键词。
* 表明你的最终目标是什么（或者不是什么）。
* 在适用的情况下，包含堆栈跟踪（stacktrace）。确保\*粘贴\*整个堆栈跟踪。不用担心占用空间 - 不用为它付费。
* 提供你正在使用的代码/配置的测试示例。
* 指出你自己为解决问题/问题所做的尝试。
* 问问自己，解决类似问题还需要什么额外信息？
* 最后：多次校对你的问题，根据需要进行重写，以便更清晰和简洁。这样做会大大提高获取帮助和找出问题所在的几率。

#### 不要做

* 将用户问题发布到开发人员的邮件列表或IRC频道。
* 在现有主题上开始新的讨论。对于新的主题，请启动一个新的线程。
* 期望即时或一次性的答案。邮件列表不是你的个人导师。
* 说的含糊不清，例如“请帮我构建一个电话系统”。没有人有时间写一个模糊的答案。
* 反复提问同一个问题。如果有人有答案，早晚你会得到它。如果你在星期五或周末发布，请等到人们度过周一的宿醉后再期望得到回复。
* 混淆你的代码或发明代码/配置片段 - 如果你在阅读这篇文章，那么你的代码/配置不是高深莫测的，也没有人会窃取它。即使你愿意付钱，也可能没有人会偷走它。
* 使用短语“它不工作”。IRC上最常见的问题是：“不工作是什么意思？”
* 尝试通过第二次重新发布“推动”你的问题。邮件列表不是一个phpBB论坛。

对于我们许多人来说，英语不是我们的母语。尽量按照英语的语法结构来表达，但不要浪费大家的时间来道歉。人们会意识到你的困难，如果需要的话会要求澄清。然而，请避免使用“短信语言”和过于随便的用语。

错误的示例：

```text
i wan 2 make cms pls cn u hlp asap
```

### 对电子邮件/回复进行格式设置

在可能的情况下，请使用内联注释，这将帮助人们确定您回复的哪部分与您上一封电子邮件相关。

错误的示例：

```text
I have checked this config and it is working fine. Can you tell me what is causing the problem?

>> (some code here)
> You are having problems because of a sofia.conf typo
>> (some code here)
> You are also having problems because of compile time errors
```

正确的示例：

```text
>> (some code here)
> 由于 sofia.conf 中的拼写错误，您遇到了问题。

我已经检查了该配置，它工作正常，这是证明；

(some code here)

> 由于编译时错误，您也遇到了一些问题。
>> (some code here)

您能指出配置中有哪个部分存在拼写错误吗？
```

### 后续行动

* 一直努力说谢谢。
* 如果您觉得有人没有理解您的请求（或者给出了无关的答案），那么重要的是向他们表达这一点，并进行进一步的澄清。
* 一直努力将您的发现公开。可以以 Django 核心工单、[djangosnippets.org](http://djangosnippets.org)、博客文章、YouTube 视频等形式呈现。（尽管有时，如果邮件列表的主题足够描述，将其放在存档中可能就足够了，这是自动完成的。）

### 五换一

像其他项目一样，如果您展示出乐于助人和贡献的意愿，其他人更有可能在将来帮助您。

虽然它没有正式宣传，但大多数核心开发人员都会参与 5 对 1 的方案。简而言之，如果您花时间处理其他 5 个工单，您可以请求核心开发人员查看您选择的任何工单。

以下准则适用：

* 您投入的每个小时，将会得到大约 3 到 5 分钟的回报。
* 核心开发人员不一定会提供修复措施，但会全力关注工单并一般性地给予更详细的反馈。
* 一般来说，至少花费 30 分钟的时间进行处理是一个好的准则。
* 如果您在一个工单的处理上花费了 30 分钟或以上的时间，您可以进行一对一的请求，但请确保在您的邮件列表请求中解释清楚（如下所示）。

以清晰简明的方式将您的请求发送到邮件列表，概述您进行的处理、采取的行动，并指明您想要的工单，例如：

```text
主题：5 对 1 请求

我花费了大约 1 小时处理以下工单的问题：

* http://jira.freeswitch.org/browse/FS-xxxx - 请求进一步信息
* http://jira.freeswitch.org/browse/FS-xxxx - 添加了评论，需要核心开发人员审查
* 等等等等

作为回报，请核心开发人员查看以下工单：

http://jira.freeswitch.org/browse/FS-xxxx
```

### 紧急请求

请记住，FreeSWITCH 社区中的所有人都是志愿者，强求得到回复不会有好结果。

如果您有紧急情况，可以考虑以下几种方式：

* 如果您有一个功能请求的票据，请在票据上悬赏并在邮件列表中发布。
* 如果您需要核心开发人员的审阅，请尝试向[anthony.minessale@gmail.com](mailto:anthony.minessale@gmail.com)发送50美元，这可能会使您获得Tony（FreeSWITCH的BDFL）的几分钟时间来评论您的票据。虽然参与5比1可能更有用。
