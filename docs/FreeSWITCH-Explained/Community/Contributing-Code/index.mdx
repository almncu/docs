## 帮助支持

登录[freenode.net](http://irc.freenode.net)的 #freeswitch 频道。即使你不是专家，只要你有一个工作正常的系统，你可能能够为刚刚开始的人提供有价值的信息。邮件列表也是一样的，如果你能回答问题，请随时回答。

## 贡献代码

欢迎贡献新功能和修复的代码。

请查看我们的文档以了解如何创建[拉请求](./Pull-Requests_3966501.mdx#about)。

与我们的开发人员在 irc.freenode.net 上的 **#freeswitch-dev** 频道交流。

我们始终需要一些[日常事务项目](./Janitorial-Tasks_3965753.mdx#-freeswitch--janitorial-tasks-)上的帮助。

如果你已经编写了代码并希望贡献，请申请[git读/写](https://confluence.freeswitch.org/display/FREESWITCH/Tips+for+using+Git)权限。

## 报告错误

报告问题并确保及时回应的最佳方法是使用问题跟踪器：<https://jira.freeswitch.org>

1) 确定你的错误是否真的是个错误。

不是寻求支持的话，你不应该提交错误报告。你可以使用_邮件列表_来请求支持。

2) 确保你的bug还没有被报告过。

在适当的问题跟踪器中进行搜索。如果找到了类似于你的bug，请检查是否有新的信息可以报告给开发人员帮助修复bug。

3) 收集关于问题的信息

为了有最好的机会修复bug，我们需要能够轻松复现导致bug的条件。

## 寻求帮助的地方

如果你有任何问题或遇到任何障碍，请与我们联系。你可以发送电子邮件到我们的[开发邮件列表](http://lists.freeswitch.org/mailman/listinfo/freeswitch-dev)。请注意，虽然你可以自由地将补丁发送到该列表用于问题或审查，但是发送到邮件列表的补丁不会被考虑纳入。你想要包含在FreeSWITCH中的补丁必须作为[Pull Requests](./Pull-Requests_3966501.mdx#about)提交。

你也可以在[freenode.net](http://freenode.net/)上找到我们，加入**#freeswitch-dev**。

在[https://cantina.freeswitch.org/vc/#/?autocall=888](https://conference.freeswitch.org/vc/#/?autocall=888)上参加每周四的 Bug Hunt Call。

最后，请随时加入我们的[每周会议通话](../ClueCon-Weekly-Conference-call_3965594.mdx#about)。许多核心开发人员经常参加会议，您将有机会在会议的开始或结束时提出您的问题。
