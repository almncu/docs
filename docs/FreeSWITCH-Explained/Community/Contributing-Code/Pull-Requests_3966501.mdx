## 使用Github进行Fork

* 在[Github](https://github.com)上创建一个新的账号。
* 在[Github](https://github.com/signalwire/freeswitch)上创建一个新的问题（issue），描述你想开发的功能或者想要解决的问题。

在[FreeSWITCH 代码仓库页面](https://github.com/signalwire/freeswitch)上，将鼠标悬停在右侧的Fork按钮上并点击。

![](/attachments/3966501/45089305.png)


## 克隆你的Fork

  
**克隆你的新Fork**

```xml
git clone https://github.com/${YOUR_USERNAME}/freeswitch.git
cd freeswitch
```

## 确保Git已设置

告诉Git你的全名和电子邮件地址，确保使用真实姓名而不是用户名。在FreeSWITCH源代码目录中运行[scripts/setup-git.sh](https://freeswitch.org/stash/projects/FS/repos/freeswitch/browse/scripts/setup-git.sh)：

```bash
./scripts/setup-git.sh 
```

## 创建工作分支

最好是为要解决的问题创建一个专用分支，这样在同步你的Fork仓库时会更容易，因为你没有在主仓库之外的分支上进行提交。

  
```xml
$ git checkout -b bugfix/fix-this-little-thing
切换到一个新分支 'bugfix/fix-this-little-thing'
```

## 提交你的更改

提交本地结果；详见[提交指南](./Commit-Message-Guidelines_6587655.mdx#about)以了解如何编写良好的提交消息：

```bash
git commit -vam "[my_module_name] This commit fixes all the things \o/"
```

请遵循提交指南中引用的模块名称\[my\_module\_name\]。

这样易于阅读提交并确定受影响的模块。

根据需要创建更多提交，以确保每个提交都代表了一个逻辑上独立的更改：

```bash
#while true; do emacs .; git commit -va; done
```

审查更改并确保您的作者名称正确：

```bash
git log -p
```

将更改推送到已派生的存储库：

```bash
git push origin
```

## 创建拉取请求

要创建您的拉取请求，请转到 GitHub 并导航到您要将更改贡献的存储库。

在“拉取请求”选项卡中，点击“New pull request”创建一个针对 FreeSWITCH 的拉取请求。

  
使用新代码更新拉取请求

如果其中一个开发人员要求您修改代码，则无需创建另一个拉取请求。

每个拉取请求都会跟踪来自派生存储库的分支，因此当您提交更多更改并推送这些更改时，拉取请求会自动更新为新的提交。

始终为每个拉取请求创建一个分支。如果开发人员要求进行更改，请在特定分支中进行更改，并推送新更改。

## 获取反馈

为增加您的拉取请求被接受的机会，请确保遵循我们的[编码指南](./Coding-Guidelines_6586862.mdx#c)和[提交消息指南](./Commit-Message-Guidelines_6587655.mdx#about)。
