## 领养者的责任

* 保持维基的最新状态。
* 在邮件列表、IRC和其他渠道上帮助他人解答有关你的模块的问题。
* JIRA上关于你的模块的票据将自动指派给你。你将率先解决错误和功能请求。
* 监控FishEye，记录他人对你的模块进行的新功能和错误修复，并在维基上记录它们。
* 你需要负责给你的模块换尿布、洗澡、洗衣服、支付医生费用、建立一个供他们上大学的基金等。

## 为什么你应该领养一个模块

* 是提高你关注的模块或模块的可见性的好方法。
* 成为该特定模块的专家。
* 提高你的FSRespect指数。
* 在你的FreeSWITCH简历上很漂亮。
* 提高你与"女士们"约会的机会。不管那是什么意思。

## 常见问题解答

### 一个模块可以被多人采纳吗？

可以。像mod_callcenter这样的较大模块将需要多人协作。一个人照顾这样的模块需要太多时间和精力。作为团队，您需要协调一致，确定需要记录什么内容，需要修复哪些错误等。

### 如果我无法继续维护一个模块会发生什么？

- 如果你正在团队中工作，请和他们讨论并告诉他们发生的情况。
- 从此页面上删除你的名字。
- 如果你是唯一一个在维护该模块的人，请知会其中一位核心开发人员。
- 向邮件列表发送邮件，并尝试询问是否有人可以接管它。
