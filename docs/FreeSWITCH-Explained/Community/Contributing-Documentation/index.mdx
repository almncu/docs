## 帮助完善文档

加入文档团队，并协助更新FreeSWITCH的文档，尤其是如果您是某个特定模块或应用程序的专家。

* [如果你还没有创建JIRA账号](https://freeswitch.org/jira/secure/Signup!default.jspa)，请先创建账号。这样你就可以访问FreeSWITCH™项目的Confluence Wiki了。

* 然后，给[Andrey Volk](https://freeswitch.org/confluence/display/~andywolk)或[Mike Jerris](https://freeswitch.org/confluence/display/~mikej)发送消息，请求加入confluence-editors组。（这有助于限制页面的破坏）

## 旧Wiki已停用

我们已将最后的页面从MediaWiki迁移到了Confluence。现在的任务是将每个页面按照Confluence的规范进行格式化，并将其移动到目录的适当位置。请注意，每个页面仍然独立存在，无论其在目录窗格中显示的样子如何，但是页面上的小部件（如“子页面”）会根据目录的层次关系来正确展示信息。

## 开始帮助格式化页面

首先仔细阅读我们的Confluence指南：[Confluence Wiki标准和指南](./Confluence-Wiki-Standards-and-Guidelines_1049056.mdx)。

许多已移动的页面仍需要重新格式化；这是开始使用FreeSWITCH文档项目的简便方法。

如何帮助进行格式和布局？有些页面可能仍需要将代码示例放置到"code macro" (<https://confluence.atlassian.com/display/DOC/Code+Block+Macro>)中，长页面可能需要在页面顶部插入适当的“Table of Contents”宏，并且基本上您可以想到的任何内容都可以使页面看起来更好。

## 额外内容

### Confluence的目的

Confluence应该是一个实时更新的文件，提供关于安装和维护FreeSWITCH的最佳实践和功能描述。使Confluence更接近书籍叙述，从大纲中删除编程内容。Confluence应始终与当前代码版本匹配。当一个分支被宣布为发布版本时，将在同一时间拍摄Confluence空间的快照，以记录代码的那个版本在那个时间存在。

### 版本控制

anthm希望文档与当前可用软件相匹配。术语“稳定版”已经被弃用。“稳定版”仅意味着开发者不再对其进行修改，而并不意味着没有错误，因此该术语并没有神奇之处。保持仅有一个与当前代码库相匹配的文档知识库与JIRA票据仅在与最新代码库进行测试后才会被考虑的思想相一致。这使得文档更加容易，因为它与软件一起演化，并且只需要更新一个知识库。
