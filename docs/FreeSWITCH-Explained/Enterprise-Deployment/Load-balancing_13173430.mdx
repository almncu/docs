# 负载均衡

负载均衡是根据某种算法将请求发送到不同的服务器的方法，从而使整个服务器的负载在每个单独的服务器上相对均衡。

这对于每秒调用数量非常高的环境非常有用，因此每个单独的服务器会看到更低数量的每秒调用。其他用途包括分配负载，以便您不会先填满一个服务器然后再转向下一个服务器，以及如果第一个服务器出现故障，您不会丢失所有呼叫。

负载均衡有几种可用的算法。其中较常见的有：

- 轮询（round-robin）：不论给定的服务器已经有多少，按顺序分配给每个服务器。这种方法不起作用，因为你可能会看到一个服务器接收到很长的调用，而其他服务器接收到相当短的调用，导致负载不均衡。
- 基于负载路由（load-based routing）：将新请求路由到负载最低的服务器。负载可以根据实际CPU利用率、已经建立的并发连接数量或其他方式来定义，以确定给定服务器的繁忙程度。
