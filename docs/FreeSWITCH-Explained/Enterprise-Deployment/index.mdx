# 企业部署

## 关于

引述下面的列表都是**虚构的作品**，截止到2008年5月14日。请阅读并了解，您必须自己进行尽职调查来扩展FreeSWITCH的规模。

“经常情况下，我的FreeSWITCH路由服务器每秒达到> 400个会话（受限评级）和> 4000个并发会话（同样受限），CPU利用率约为20％，内存利用率约为30％。这是在一台戴尔1950机器上，配备双物理E5335 2Ghz四核CPU和4GB内存。”任何SPS和并发限制都是为了保护上游合作伙伴，我至少有3次的情况，其中大型Tier 1 / Tier 2英特尔和国内美国长途运营商都请求宽限，有一次打电话说“你能否请慢一点，不要把我的{Censored} SBC搞垮了”，这两家供应商我知道的确实是Veraz和AcmePacket。” -我们用户群中的某个人（当我们接近1.0版本发布时）。

FreeSWITCH™拥有很高的性能，但是有时候一个单独的服务器的性能还不够，要么是因为你需要故障转移，负载均衡处理多个服务器无法承载的负载，要么是因为需要某种容错机制，这是一个单独的服务器无法提供的。

您可以采用多种方法来实现这个目标。以下是一些选择。强烈建议您了解这里使用的概念，以及为什么它适用于您的整个系统设计，不要只是随意选择一个，因为有人建议它适用于他们。为了最大化性能，您需要创建适合特定环境的解决方案，适合一种环境的解决方案可能对另一种环境完全不适合。
