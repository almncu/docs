## 为什么要使用 Xen？

* [负载管理](https://wiki.freeswitch.org/wiki/Load%5Fmanagement "负载管理")
   * 将高负载进程迁移到空闲服务器
   * 将空闲进程迁移到繁忙服务器
* 降低成本
   * 通过更好地利用空闲的CPU时间购买更少的服务器
   * 由于大多数服务器大部分时间都处于空闲状态，因此不使用它几乎是对服务器的浪费
* 增加正常运行时间
   * 在不关闭运行中的应用程序进行维护或升级硬件的情况下，将其迁移到另一台服务器
* CPU的QoS（Quality of Service）
   * 确保您的虚拟机没有因CPU进程不足而受影响
* CPU时钟不会在虚拟机内随意浮动。
   * 如果要运行实时VoIP应用程序，则严格的定时非常重要。

## 示例

您在5个物理系统上运行着 Xen，每个 Xen 服务器创建30个虚拟系统。每个虚拟系统都运行 FreeSWITCH™ 和一个会议应用程序。当开始使用时，您注意到一些虚拟服务器的通话量比其他服务器大得多，因此您将活动虚拟系统移动到不同的物理系统上以分担负载，并与拥有更多空闲CPU的物理系统共享，而无需中断通话。

## 范围

该文档并不涵盖XEN部署的所有方面，包括但不限于正确的网络规划，这可能涉及到多个私有网络（带有多个网卡）用于机器间的通信；[集群文件系统](https://wiki.freeswitch.org/wiki/Enterprise%5Fdeployment%5FClustered%5FFile%5FSystems "Enterprise deployment Clustered File Systems")和系统调优，这取决于您服务器的具体用途。

## 注意事项

在某些64位Xen DomUs上，如果使用-hp（高优先级）开关来运行FreeSWITCH，可能会导致内存管理问题。这会导致随机的故障。建议在Xens环境下运行时禁用-hp参数。
