# 模块

待办事项 **如何添加模块**。每个模块通常描述了如何添加它们，但没有通用的描述。（在 [mod\_shout](./mod_shout_3965531.mdx#21-play) 的“第2章 加载 `mod_shout`”中将是一个不错的模板候选。）

待办事项 没有提到 `modules.conf.xml` 或 `pre_load_modules.conf.xml`（对于后者，在 [Debian 9 Stretch (Obsolete)](../Installation/Linux/Deprecated-Installation-Instructions/15139022.mdx#about) 中只有一个注释）。

参见 [XML 模块配置](./XML-Modules-Configuration_13173211.mdx#xml-modules-configuration) 中的待办事项。

## 关于

通过添加执行特定任务的模块，可以扩展FreeSWITCH的功能，无论该任务是简单还是复杂。这些模块可以根据其各自的模块页面上的标签分为广泛的类别。

## 配置

### 将模块设置为“critical”

如果由于某种原因，您需要防止FreeSWITCH在特定模块启动时加载失败后完全停止，则可以使用“critical”属性：

```xml
<load module="mod_really_important" critical="true"/>
```

如果模块 _mod\_really\_important_ 加载失败，则FreeSWITCH将中止。

### 将模块文件定位于不同路径

如果您使用商业模块 `mod_com_g729`，则应将其移动到一个单独的文件夹中，如 `mod_com`，并设置您的条目如下。这样，将从 freeswitch 文件夹中删除文件夹 `lib` `bin` `mod`，并执行 `make install`，而不会丢失您的 `mod_com_g729`。

```xml
<load module="mod_com_g729" path="/opt/fs/mod_com"/>
```
