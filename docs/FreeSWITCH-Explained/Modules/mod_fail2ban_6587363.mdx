# mod_fail2ban 

## 关于

将所有入站注册尝试和失败的注册记录到日志文件 "/usr/local/freeswitch/log/fail2ban.log"（可配置）。使用此模块进行fail2ban，而不是使用标准的FS日志，优势在于无需将服务器日志级别保持为冗长。

## 作者

在2012年4月20日星期五下午7:17发布于[邮件列表](http://lists.freeswitch.org/pipermail/freeswitch-users/2012-April/083115.html)

大家好，我为Fail2ban应用程序编写了一个FreeSWITCH模块。
它当前将所有入站的注册尝试和失败的注册记录到日志文件 "/usr/local/freeswitch/log/fail2ban.log"（可配置）。使用此模块进行fail2ban，而不是使用标准的FS日志，优势在于无需将服务器日志级别保持为冗长。

## 安装说明

* 源代码：<https://github.com/kyconquers/freeswitch-mod%5Ffail2ban>
* `git clone <https://github.com/kyconquers/freeswitch-mod%5Ffail2ban.git>`
* 将配置文件放入 autoload\_configs 目录，并根据需要进行调整。
* 进入目录并执行 `make && make install`
* 手动加载：在 fs\_cli 中，输入 `load mod_fail2ban`
* 自动加载：在 modules.conf.xml 中，添加 `mod_fail2ban`

## 提示

如果 make 失败，出现以下错误：

* `` make[1]: *** No rule to make target `/usr/local/src/freeswitch-mod_fail2ban/freeswitch-mod_fail2ban.c', needed by `[freeswitch-mod_fail2ban.la](http://freeswitch-mod%5Ffail2ban.la)'.  `` `Stop`

请将 git 克隆到您的 freeswitch 源代码目录中，即 /usr/local/src/freeswitch/src/mod/applications/mod\_fail2ban，并从该目录执行 make：

* cd /usr/local/src/freeswitch/src/mod/applications
* git clone <https://github.com/kyconquers/freeswitch-mod%5Ffail2ban.git> mod\_fail2ban
* make && make install
