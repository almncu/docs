## 0\. 关于

将通道放置在交换机中的“等待中”，而不是电话中。

将呼叫停留在“停泊位”上并不提供保持音乐或其他媒体。

Park（停泊）是一个将呼叫暂时置于待定状态的方法，直到你

* 将其桥接（使用[mod\_dptools：bridge](./6586484.mdx#45-user)或在[mod\_commands](../mod_commands_1966741.mdx#--)中使用`uuid_bridge`）  
或者
* 转接（通过[mod\_dptools：transfer](./6586616.mdx#about)或在[mod\_commands](../mod_commands_1966741.mdx#--)中使用`uuid_transfer`）。

可以选择多种不同的选项，包括：

* 在等待与某人通话时，设定呼叫者的位置，以避免通道被挂断。
* 通用的“等待”机制，直到你可以将呼叫与正确的人员连接。

请注意，要检索已“停泊”（parked）的呼叫，您需要将其[桥接](./6586484.mdx#45-user)到它们或将呼叫转接到有效位置。

## 1\. 使用方法

### 1.1 从拨号计划中使用

```xml
<action application="park"/>
```

请注意，park应用程序不接受任何参数，因此在定义中可以省略data属性。
