# mod_dptools: bridge_export 

## 关于

在任何桥接中导出通道变量。该应用程序与“export”不同之处在于它可以与\*任何\*类型的桥接一起使用，而不仅仅是从拨号计划中调用的桥接。例如，bridge\_export将在uuid\_transfer上导出变量，而export则不会。

## 用法

```xml
<action application="bridge_export" data="foo=bar"/>
```

执行桥接应用程序后，您应该能够在a-leg和b-leg中看到以下内容：

[variable_bridge_export_vars] = [foo]  
[variable_foo] = [bar]

与[export](./6586592.mdx#about)类似，要将导出限制为仅适用于B leg，请使用“nolocal:”标志，如下所示：

```xml
<action application="bridge_export" data="nolocal:foo=bar"/>
```
