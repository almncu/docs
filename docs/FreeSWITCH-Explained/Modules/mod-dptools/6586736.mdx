# mod_dptools: delay_echo

## 关于

将呼叫通道置于延迟音频回放模式。它简单地在指定的延迟时间\[毫秒\]后返回所有发送的音频，包括语音、DTMF等。通常用于检查RTP音频路径是否双向工作。正常的回声应用程序在使用扬声器电话进行测试时可能失败。

### 示例

#### 来自拨号计划

```xml
<action application="delay_echo" data="1000"/>
```

这个示例在延迟1秒（1000毫秒）后重复播放接收到的任何音频。
