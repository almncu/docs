## 0. 关于

挂断一个通道，并且可选择提供一个原因代码。

## 1. 使用方法

```xml
<action application="hangup" data="USER_BUSY"/>
```

如果在桥接之后使用hangup，FreeSWITCH™会用从桥接应用程序接收到的原因覆盖。你可以通过设置sip\_ignore\_remote\_cause=true来避免这种情况。

## 2. 故障排除

以下是[Bao Nguyen](https://freeswitch.org/confluence/display/~nttranbao)分享的技巧。

### 2.1 调用hangup后无法断开一个通道

你是否看到了FreeSWITCH™向呼叫者发送的`BYE`消息及其200 OK响应？如果是的话，那么问题可能出在你的ITSP或呼叫者所在的服务提供商。

当B方终止通话后，如果FreeSWITCH™没有断开A通道，也可以检查 [hangup_after_bridge](../../../Channel-Variables-Catalog/hangup_after_bridge_16352708.mdx#-freeswitch--hangup_after_bridge-) 变量。
