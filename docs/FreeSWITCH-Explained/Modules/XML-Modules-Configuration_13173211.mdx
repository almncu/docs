## XML模块配置

modules.conf.xml是[freeswitch.xml](../Bit-Bucket/For-Consideration/FreeSWITCH.xml_13173070.mdx#default-configuration-file)中“配置”部分中存在的配置实体。它指定要加载的模块以及加载顺序。

要加载一个模块，它需要在系统上被编译和安装到正确的位置。如果您不知道如何编译/安装特定的模块，请阅读[安装指南](../Installation/index.mdx#about)。

只有满足上述条件时才会加载模块。只是在freeswitch.xml中包含一个模块的配置不会导致该模块被加载。

举个例子：

* 我们希望加载syslog以便在生产过程中出现故障时进行日志记录。
* 我们将加载mod\_cdr以便从系统中获取呼叫数据记录（用于计费非常好）。
* 我们将使用mod\_event\_socket以便能够获取所有连接的用户代理的事件（并随后对它们进行控制）。
* 我们将使用XML拨号计划以简化操作。
* 最后，我们加载spidermonkey以便编写JavaScript IVR应用程序，以及加载mod\_cepstral以获取TTS功能。

其他模块都很常规，不应该移除。
