# 群铃组

单击此处展开目录

**群铃组** 是 PBX 电话术语，指的是一组物理电话，所有电话同时响铃，以响应拨打到来电号码（也称为“DID”，是“直接呼入拨号”的首字母缩写）的电话。

在端点之间加上逗号“，”，就会引起“群铃组”的行为，这意味着所有分机（电话）将同时响铃。

注意：

* 如果一个或多个电话在一个目录号上注册（启用了多重注册），当拨打该目录号且号码之间用逗号“,”分隔时，只有一个电话会响铃。如果希望所有电话在同一个分机号上同时响铃，请使用冒号下划线冒号“:_:”代替逗号“,”。

这与猎头组相关，猎头组将呼叫转接到“指示号码” DID，并按顺序逐个响铃。通常，企业会宣传一个电话号码，但拥有多个未公开的号码，以便在任何忙线/占线的情况下逐个转接。通过这种方式，拨打“主要号码”的电话可以同时响铃一系列电话。

在端点之间加上竖线“|”，就会引起“猎头组”的行为，这意味着电话列表将依次顺序响铃。您还可以设置超时变量。可能还可以为每个分机（在长猎头组列表中）设置不同的超时，通过方括号指定每个分机的超时时间。

如果分机不经常更改，请使用使用竖线分隔的端点列表。

将此行放入`${conf_dir}/dialplan/public/in.3015551212.xml`中，将导致拨打到 301-555-1212 的来电同时在企业 A 的两个分机上响铃，x1021 和 x1022：

```xml
<action application="bridge" data="sofia/multi.domain-a.com/1021%${domain_name},sofia/multi.domain-a.com/1022%${domain_name}" />
```

不要使用"user/"通道来形成组呼拨号字符串；而是像上面的示例中那样，使用百分号 % 替代域名前的 @ 符号。
