## auto_hunt

将 auto_hunt 设置为 "true" 将改变拨号计划扩展的正常顺序处理。 auto_hunt 会导致拨号计划'跳转'到特定的**扩展名**，而不处理其他的扩展。为了使其起作用，目标号码和扩展名必须相同。条件仍然需要匹配，但扩展名是操作元素。

在下面的示例中，没有办法不使用 auto_hunt 而达到扩展 333。

**用法：** 在 vars.xml 文件中：

```xml
<X-PRE-PROCESS cmd="set" data="auto_hunt=true"/>
```

**示例：**

```xml
<extension name="do_xfer">
  <condition field="destination_number" expression="^.*$">
    <action application="set" data="auto_hunt=true"/>
    <action application="transfer" data="333"/>
  </condition>
</extension>

<extension name="333">
  <condition field="destination_number" expression="^333$">
    <action application="info"/>
  </condition>
</extension>
```
