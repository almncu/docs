## accountcode

账户代码主要是一个任意值，可以按每个通话分配。_accountcode_ 的一个重要特性是如果其值与 **cdr\_csv.conf.xml** 中定义的CDR CSV模板之一匹配，那么在生成CSV CDR时将使用该CDR模板。

#### **用法:**

```xml
<action application="set" data="accountcode=custom"/>
```

#### **由以下模块实现:**

| **模块名称**                                   | **源文件**         | **最后修订** |
| --------------------------------------------- | ----------------- | ------------ |

## copy_xml_cdr

将另一个通话的XML CDR复制到该通话的CDR中。例如，A通话的CDR将包含一个名为**b\_leg\_cdr**的变量，其内容是B通话的URL编码XML CDR数据。必须在B通话中设置此变量，因此在拨号字符串中使用**{copy\_xml\_cdr=true}**或使用**export**而不是set。

#### **用法:**

```xml
<action application="bridge" data="{copy_xml_cdr=true} user/${dialed_extension}@${domain_name}"/>
```

#### **实现者:**

| **模块名称** | **源文件**              | **最近修订** |
| ------------- | ------------------------ | ------------ |
| core          | switch\_ivr\_bridge.c    | 8955         |

## failed_xml_cdr_prefix

如果在A通话上设置了这个参数，并且任何失败的B通话都生成一个完整的XML CDR报告并将其设置为一个变量，这个变量包括在分叉拨号中。

假设您尝试拨打`sofia/profile/a@xxxxxxx,sofia/profile/b@xxxxxxx`

而且它在完全失败之前，您将`failed\_xml\_cdr\_prefix`设置为"bad\_call"。

那么你最终将得到`${bad\_call\_1}`和`${bad\_call\_2}`，它们各自是一个包含所有变量等的完整XML报告。

#### **用法:**

```xml
<action application="set" data="failed_xml_cdr_prefix=failinggw" />
```

#### **另请参阅:**

* Maillist message [\[1\]](http://www.mail-archive.com/freeswitch-dev@lists.freeswitch.org/msg00341.html)

| 核心         | switch_ivr_originate.c | 8730          |

## process_cdr

**指示如何处理CDR记录。**

可以未定义或设置为"false"、"true"、"a_only"、"b_only"

* false - 表示不处理记录。
* true - 或未定义，表示默认行为，即处理所有CDR记录。
* a_only - 表示仅处理呼入通话的CDR记录。
* b_only - 表示仅处理呼出通话的CDR记录。

_此变量无条件导出_

####   
**用法:**

&lt;action application="set" data="process_cdr=a_only"/>

#### **实现者:**

| **模块名称** | **源文件**              | **最后修改** |
| ------------ | ------------------------ | ------------- |
| 核心         | switch_core_session.c        | 7214          |
| 核心         | switch_core_state_machine.c | 7212          |
