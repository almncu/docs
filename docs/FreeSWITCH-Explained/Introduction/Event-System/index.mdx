## 0\. 关于

FreeSWITCH是事件驱动的，它会响应和发出事件。[事件](./Events_32178330.mdx#0-关于)是系统上发生任何活动的结果，并由FreeSWITCH的[核心和模块](../index.mdx#3-架构)以及外部来源生成。事件系统也使得构建[event socket](../Glossary_13173966.mdx#event-socket)应用程序成为可能。

您可以在[fs\_cli](../../Client-and-Developer-Interfaces/1048948.mdx#or)中快速查看事件系统的运行情况：

```xml
$ fs_cli
freeswitch@your_host> /event ALL
```

现在，您只需开始发起呼叫、发出命令或者仅仅等待，事件就会开始涌入。

有关事件的全面列表，请参阅 [事件列表](./Event-List_7143557.mdx#nat)。

## 1\. 事件

事件系统允许外部软件程序作为监听器进行操作。在FreeSWITCH系统内发生的几乎所有事件都会生成某种类型的**事件消息**。这些事件可以被外部实体观察到。这类似于常见消息队列软件解决方案中使用的发布/订阅（或"pub-sub"）系统，但它专为FreeSWITCH事件量身定制。

事件系统是双向的：除了允许外部程序监听事件外，外部程序还可以向FreeSWITCH发送事件。您可以实时地从自己的程序中发送和/或接收事件。这种组合使您能够以几乎任何您可以想象的方式使用FreeSWITCH。
