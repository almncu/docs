## Liverpie

Liverpie (语言无关的 IVR 代理) 是一个免费的小型软件，用 Ruby 编写，与 FreeSWITCH 一边通信，与任何 Web 应用程序（不论语言、平台等）另一边通信。它将 FreeSWITCH mod\_socket 对话转换为 HTTP 对话（在 HTTP 头中嵌入各种参数），因此您可以编写自己的支持 HTTP 的有限状态机，并通过 Liverpie 将其连接到 FreeSWITCH。此外，请注意，Liverpie 期望以 YAML 格式接收响应，因此如果对 Liverpie 执行翻译方面感到舒适，您可以免去提供 XML 的痛苦。

在这里找到 Liverpie：[www.liverpie.com](http://www.liverpie.com)

## Telnet 客户端

使用 Telnet 连接到端口 8021，并输入 "auth ClueCon" 作为密码进行身份验证。从这里，您可以发送此文档中列出的任何命令。请注意，您需要在命令之后提供两个空行以便进行处理。

## Python 库

* 在FreeSWITCH源代码的[scripts/python/freepy](https://freeswitch.org/stash/projects/FS/repos/freeswitch/browse/scripts/python/freepy)目录中可以找到一个基于[Twisted](http://www.twistedmatrix.com)的客户端和示例。
* [eventsocket](https://github.com/fiorix/eventsocket): 这是一个替代的Twisted协议，支持入站和出站方法和示例。
* [PySWITCH](http://sourceforge.net/projects/pyswitch/?source=directory): 这是另一个面向Python和Twisted程序员的库。它对FreeSWITCH API、bgapi和Dialplan工具有广泛的支持。
* [greenswitch](https://github.com/EvoluxBR/greenswitch): 这是一个使用[gevent](http://www.gevent.org/index.html) greenlets实现的事件套接字协议。它已经投入生产，每天处理数百个呼叫。
* [switchio](https://github.com/friends-of-freeswitch/switchio): 这是一个由[asyncio](https://docs.python.org/3.6/library/asyncio.html)驱动的集群控制库，利用了现代Python的新原生[coroutines](https://docs.python.org/3.6/library/asyncio-task.html)技术。它还附带了一个完整的自动拨号器，最初用于压力测试。

##  Javascript / Node.js库

* [GitHub](https://github.com/englercj/node-esl)和[npm](https://npmjs.org/package/modesl)上均可获取Node-esl模块。该模块提供对入站和出站连接的支持，用于多个出站连接的服务器助手，并实现了[Event Socket Library](https://wiki.freeswitch.org/wiki/Event%5FSocket%5FLibrary "Event Socket Library")接口。示例代码可以在源代码的[examples directory](https://github.com/englercj/node-esl/tree/master/examples)中找到。
* Node.js esl模块（[可通过npm获取](http://search.npmjs.org/#/esl)）提供了客户端和服务器实现。代码在[github上](https://github.com/shimaore/esl)，并且有相应的[文档](http://shimaore.github.com/esl/esl.html)。示例代码包括：[使用CouchDB存储的语音信箱](https://github.com/shimaore/ccnq3/tree/master/applications/voicemail/node)和[CNAM注入](https://github.com/shimaore/ccnq3/blob/master/private/cnam-client/node/cnam-client.coffee)（简短示例展示了如何使用异步网络查询设置变量）。
* [Node.js ESClient](https://github.com/demchenkoe/njs-freeswitch-esc)
* 使用出站ESL的IVR示例：<http://freeswitch-users.2379917.n2.nabble.com/Javascript-Outbound-Event-Socket-Linger-Command-td7530943.html>

## .NET客户端库

支持Windows环境下构建。请参阅libs/esl/managed/managed\_esl.201x。

示例应用程序中包含了一个名为ManagedEslTest的样例应用程序。它演示了如何启动一个简单的入站示例，并展示了其他模式。

所管理的部分是平台中立的，但是ESL SWIG 包装器（托管解决方案中的ESL项目）是平台相关的，必须针对所需的平台进行构建。您还必须先构建主要的FreeSWITCH 项目，因为ESL库是从那里包含的，否则您将收到esl.lib未找到的错误。
