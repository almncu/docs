# SSL, SSLv2, SSLv3 

 

## 关于

SSL加密的SSLv2和SSLv3版本已被破解（SSLv1从未发布）。因此，在使用被破解的加密方式没有太多意义之下，建议您将SSL支持从客户端和服务器上移除。

* 2[SSL证书](#ssl-certificates)

## 在FreeSWITCH中禁用SSLv3和SSLv2

我们已经提供了正确的配置很长时间了，但您可能需要检查您的设置。

您不希望在tls-version参数值中包含sslv2或sslv3。

编辑/usr/local/freeswitch/conf/vars.xml（或如果您是通过软件包来安装的，则为/etc/freeswitch/vars.xml），确保以下行如下：

&lt;X-PRE-PROCESS cmd="set" data="sip\_tls\_version=tlsv1,tlsv1.1,tlsv1.2"/>

然后检查所有SIP配置文件是否使用了相同的值。可以从全局变量中获取值，如下所示：

&lt;param name="tls-version" value="$${sip\_tls\_version}"/>

## SSL证书

证书与使用SSL作为加密方法无关，“SSL证书”只是旧的称呼安全证书（因为它曾被SSL使用，但TLS也可以使用，没有问题，都是同一个证书）。

这里没有什么可看的。
