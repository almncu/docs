#### EyeBeam TLS设置

（在Windows上测试，使用EyeBeam 1.5.19 build 47739）

* 在帐户属性中转到**安全**选项卡。选择**TLS**作为信令传输方式。
* 获取cafile.pem文件的副本并将其重命名为cafile.crt。在“受信任的根证书”存储中安装证书（在文件浏览器中双击）。

如果出现“证书验证失败”等错误，则说明操作不正确。

如果出现“证书名称不匹配”的错误，那么服务器的证书名称与您在EyeBeam中输入的域名不匹配。是的，EyeBeam也会检查这一点。Phonerlite可以不进行检查，并且它是一个漂亮便携的软件（但也有一些自身的问题-请参见上文。希望这些问题能很快得到修复）。

如果FS关闭或由于其他原因，EyeBeam无法注册，有时会显示证书问题而不是注册失败。

* **注册：**可用
* **来电显示：**可用

* **呼叫等待：**可用
* **转接电话：**未知

* **TLS/SRTP：**可用（？）

要启用TLS，请将证书加载到Windows证书存储中，并在证书选项卡下勾选“检查远程方的证书”和“加载Windows CA”复选框。在网络选项卡下，选择“Preferred Connection Type”为TLS。 （将cafile.pem重命名为cafile.crt，在资源管理器中双击并安装到根证书存储中，信任发布者不起作用 - 即使选择了TLS，它仍然只能进行TCP连接，除非将其放入根证书存储中）（我无法让PhonerLite工作使用TLS，握手开始但未完成注册。在20次尝试中，它只有一次成功！但不久之后，即使没有做任何更改，它再也无法工作。请在此处发布任何使其工作的详细信息。谢谢。）

要启用RTP，请在编解码器选项卡下选择SRTP和SAVP。

这是一个能够进行SIP-TLS + SRTP呼叫的免费软电话！

该软件完全便携，您还可以将其放在USB驱动器上。如果不更改设置中的端口号，当phonerlite已经打开时，同一台机器上的FreeSWITCH（Windows）无法使用端口5060，除非先启动它。

请注意：如果服务器不支持TLS或无法工作，软电话将自动使用不安全的TCP连接，这是不好的！请在他们的论坛上建议修复此问题。建议的人越多，他们增加强制TLS选项的可能性就越大。

还有，我发现了一些与“不使用来自远程端的证书检查”有关的问题，我理解为不检查证书的有效性（如果它是由Windows证书存储中的任意一个根证书签名的），但仍然会执行TLS协议。它确实执行了TLS协议，但我无法完成握手过程。似乎与端口有关的问题。（在这种情况下未回退到TCP）如果有人知道如何解决这个问题，或者有其他看法，请在这里发帖！当选择检查远程证书时，无论是否选择加载Windows CA，都会回退到TCP

#### Sip NOTIFY Flood攻击

Zoiper经典版本低于v3将每1-5毫秒发送一个通知包。这是一系列小包的持续流，会淹没交换机以及沿途的几乎所有路由器（每个客户端大约4.5mbs）。作为解决方法（感谢支持人员Joachim！），您可以禁用消息等待指示。此问题在Zoiper Communicator的v3版本（也称为Zoiper 3）中已解决。（2013.09.04）

### Zoiper商业版2.10和TLS/SRTP

自3065版本的库起，Zoiper商业版也可以在Linux上使用TLS和SRTP（在Ubuntu Gutsy上进行了测试）。

如何启用：

- 在选项中勾选扩展设置
- 在SIP选项中设置端口为5061，并选择来自FreeSWITCH服务器的TLS证书（将cafile.pem复制到您的本地目录）
- 在您的SIP帐户设置中选择“使用TLS和SRTP传输”
