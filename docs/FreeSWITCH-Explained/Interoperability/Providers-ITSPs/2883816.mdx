# 供应商号组（英国）

## [numbergroup.com](http://www.numbergroup.com)

最后更新日期：2011年3月7日

使用SIP URI进行DID功能正常。通过SIP中继进行DID功能正常。外拨功能正常。

用户名只包含用户部分，不包括域名。

请注意，对于呼入呼叫，呼叫通常会命中目标号码为用户名的拨号计划。设置extension为auto_to_user可以避免这个问题，呼叫将以auto_to_user的URI形式到达，Sofia将自动从To头部获取目标号码。

默认情况下，NumberGroup支持G.711 A/U编解码器。要启用对G.722、Speex和GSM等窄带编解码器的支持，请进入您的中继配置，将"Voice Compression"更改为"Yes"。

```xml
  <include>
        <gateway name="numbergroup">
          <param name="realm" value="sip.numbergroup-services.com"/>
          <param name="username" value="USERNAME"/>
          <param name="password" value="PASSWORD"/>
          <param name="register" value="true"/>
          <param name="expire-seconds" value="3600"/>
          <param name="retry-seconds" value="3600"/>
          <param name="caller-id-in-from" value="true"/>
          <param name="extension" value="auto_to_user"/>
          <param name="caller-id-type" value="pid"/>
        </gateway>
      </include>
```
