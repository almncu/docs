# 提供者个人网络电话（德国）

## [www.personal-voip.de](http://www.personal-voip.de)

最后更新日期：2013年12月17日

可用于呼入和呼出SIP通话。

请注意，[personal-voip.de](http://personal-voip.de)希望您使用SIP ID作为用户名，而不是SIP号码！

还请注意，联系人必须通过 &lt;sipID>@[sip.personal-voip.de](http://sip.personal-voip.de) 发送。因此，扩展参数 extension-in-contact 必须设置为 true，并且扩展参数必须为空，或者与用户名字段的值相同！

```xml
<include>
  <gateway name="personal-voip">
    <param name="username" value="你的一个SIP ID"/>
    <param name="password" value="你的密码"/>
	<param name="extension" value="注释掉此参数或将其设置为与用户名字段中的SIP ID相同的值。不要在此处添加其他内容！"/>
	<param name="ping" value="如果您使用的是路由器且遇到问题，请将此参数设置为较小的值。否则请将其注释掉"/>
	<param name="realm" value="personal-voip.de"/>    
	<param name="proxy" value="sip.personal-voip.de"/>
	<param name="register" value="true"/>
	<param name="extension-in-contact" value="true"/>
	<param name="caller-id-in-from" value="false"/>
  </gateway>
</include>
```

如果您发现对于呼出通话，没有显示正确的来电者ID：

由于某种原因，[personal-voip.de](http://personal-voip.de)似乎忽略了有效呼叫者ID号码（effective\_caller\_id\_number）和源始呼叫者ID号码（origination\_caller\_id\_number），而是使用有效呼叫者ID名称（effective\_caller\_id\_name）和源始呼叫者ID名称（origination\_caller\_id\_name）字段的值。如果名称字段的值仅为数字，则会显示正确的号码。否则，将使用您电话号码块中的第一个号码。
