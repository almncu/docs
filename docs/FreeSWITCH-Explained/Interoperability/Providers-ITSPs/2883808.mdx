# 美国提供商nexVortex

## [nexVortex.com](http://nexvortex.com)

最后更新日期：2011年1月17日

示例外部配置文件:

```xml
<include>
   <gateway name="nexvortex">
      <param name="username" value="您的用户名"/>
      <param name="password" value="您的密码"/>
      <param name="from-user" value="在这里填写您的来电号码！" /> 
      <param name="proxy" value="px1.nexvortex.com"/>
      <param name="realm" value="px1.nexvortex.com"/>
      <param name="expire-seconds" value="3600"/>
      <param name="register" value="true"/>
      <param name="ping" value="600"/>
      <param name="caller-id-type" value="pid"/>
   </gateway>
</include>
```

NexVortex不使用Remote-Party-ID，因此参数名为 "from-user"。

当您使用 "from-user" 时，这将更改您的用户注册，以便DID成为注册用户，因此您必须将DID路由设置为指向DID，而不是NexVortex用户名在NexVortex呼叫路由中。
