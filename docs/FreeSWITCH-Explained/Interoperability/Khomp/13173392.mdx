# Khomp/命令

## 关于

本文档介绍了关于Khomp命令的信息。

单击此处展开目录：

* 1 [Khomp/命令](#khompcommands)

* **khomp channels disconnect**: 断开一个或多个通道。此命令直接发送消息到相关卡的物理通道，请求断开连接。请谨慎使用；
* **khomp channels unblock**: 解锁因输入或输出而被阻塞的通道。仅适用于通过E1进行数字标牌；
* **khomp clear links**: 清除链路上的错误计数器；
* **khomp clear statistics**: 清除通道连接的统计数据，或特定通道的统计数据；
* **khomp get**: 获取Khomp终端的选项数量；
* **khomp kommuter**: 启用或禁用通过USB连接到此设备上的交换机。仅在配置"kommuter-activation"设置为"manual"时可用；
* **khomp kommuter count**: 获取通过USB连接到此设备上的交换机数量；
* **khomp log console**: 设置控制台日志选项；
* **khomp log disk**: 设置日志记录选项到磁盘；
   * **khomp log console**和**khomp log disk**还有辅助选项**No**，用于反转消息的选择，以及**just**，用于概括选择。示例：
      * **khomp log disk just commands events** (仅启用对命令和事件记录到磁盘)；
      * **khomp log disk no commands** (禁用对发送给板卡的命令记录到磁盘)；
      * **khomp log disk warnings** (还启用对终端警告记录到磁盘)；
   * 有关**log**命令的更多选项信息，请输入："**help khomp log disk**"或"**help console log khomp**"。
* **khomp log rotate**: 旋转Endpoint的日志文件；
* **khomp log status**: 显示当前正在写入磁盘和显示在控制台上的日志消息；
* **khomp log trace isdn**: 启用ISDN信令调试；
* **khomp log trace k3l**: 启用底层API K3L调试；
* **khomp log trace r2**: 启用底层信令R2/MFC调试；
* **khomp reset links**: 发送重置命令到特定卡的特定**E1**；
* **khomp revision**: 显示Endpoint的版本号和修订号；
* **khomp select sim**: 选择SIM卡，适用于KGSM板卡；
* **khomp send command**: 直接发送API K3L命令到板卡（仅用于调试，如果使用不当可能会影响系统的稳定性）；
* **khomp send raw command**: 直接发送命令到DSP板卡（仅用于调试，如果使用不当可能会影响系统的稳定性）；
* **khomp set**: 设置Endpoint Khomp的各种选项；
* **khomp show calls**: 显示呼叫的状态，还可以列出特定的通道或板卡；
* **khomp show channels**: 显示Khomp通道的状态，还可以列出特定的适配器；
* **khomp show links**: 显示可用的E1链路状态；
* **khomp show statistics**: 显示通道连接的统计数据，或特定通道的统计数据；
* **khomp sms**: 使用KGSM通道发送给定号码的短信；
* **khomp summary**: 打印系统板卡及其功能的摘要信息；
