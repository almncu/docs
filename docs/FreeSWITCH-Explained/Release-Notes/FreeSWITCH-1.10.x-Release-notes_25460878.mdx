## 重大变更

* **1.8 -> 1.10.0**: **`pgqsql`从FreeSWITCH核心中移出，成为了自己的模块`mod_pgsql`**。请参见底部的**1.10.0版本（发布日期：2019年8月5日）**。
* **1.10.3 -> 1.10.4**: **删除了SpanDSP和sofia-sip模块**。请参见下面的**1.10.4版本（发布日期：2020年8月5日）**。
* **1.10.6 -> 1.10.7**: **默认行为更改：在恢复时不跳过通告类型应用程序，在SRTP错误时不挂断呼叫，每秒最大接收请求数为1000，auth-messages和auth-subscriptions默认启用，freeswitch.log日志行前缀修改：会影响fail2ban的日志行匹配，直至在fail2ban中进行类似修改（<https://github.com/fail2ban/fail2ban/issues/3143>）。**请参见下面的**1.10.7版本（发布日期：2021年10月24日）**。
* **1.10.9 -> 1.10.10**:
**已删除**
**`SWITCH_DECLARE(switch_status_t) switch_curl_process_form_post_params(switch_event_t *event, switch_CURL *curl_handle, struct curl_httppost **formpostp);`**
**请改用**
**`SWITCH_DECLARE(switch_status_t) switch_curl_process_mime(switch_event_t *event, switch_CURL *curl_handle, switch_curl_mime **mimep);`**

**请参见**<https://github.com/signalwire/freeswitch/pull/2175/files>**了解详情。**

移动: pgsql从核心中移出，成为独立的模块（mod\_pgsql）

原生的PostgreSQL（`pgsql`）从FreeSWITCH核心中移除，放入了自己的模块（`mod_pgsql`）！请仔细阅读：[FS-11880](https://freeswitch.org/jira/browse/FS-11880?src=confmacro) - [Authenticate](https://freeswitch.org/confluence/plugins/servlet/applinks/oauth/login-dance/authorize?applicationLinkID=c4b15fa3-ad57-3c05-af05-8c167cb92266)查看问题详情

要点：您必须**不要**在 `modules.conf.xml` 中加载 [mod\_pgsql](/confluence/pages/createpage.action?spaceKey=FREESWITCH&title=mod_pgsql&linkCreation=true&fromPageId=25460878)，而是在 [pre\_load\_modules.conf.xml](../Configuration/Configuring-FreeSWITCH/index.mdx#modulesconfxml) 中加载，它是一个额外的XML文件，同样位于 `conf/autoload_configs/` 目录中。

**pre\_load\_modules.conf.xml**

```xml
<configuration name="pre_load_modules.conf" description="Modules">
  <modules>
    <!-- 数据库 -->
    <!-- <load module="mod_mariadb"/> -->
    <load module="mod_pgsql"/>
  </modules>
</configuration>
```

有关更多内容，请参阅[此Stackoverflow主题](https://stackoverflow.com/questions/57634856/trying-to-connect-to-a-postgresql-db-via-pgsql-dsn-returns-err-unable-to-ope)。
