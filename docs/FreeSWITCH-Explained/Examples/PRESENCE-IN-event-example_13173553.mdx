# PRESENCE IN 事件示例

## 关于

以下事件可以通过[Event Socket](../Modules/mod_event_socket_1048924.mdx#11-acl)发送，以使BLF灯执行某些操作。已在snom 360上进行了测试。

单击此处展开目录

* 1 [闪烁灯:](#闪烁灯)
* 2 [打开灯:](#打开灯)
* 3 [关闭灯:](#关闭灯)

### 闪烁灯:

```xml
sendevent PRESENCE_IN
proto: sip
from: 801@example.com
login: 801@example.com
event_type: presence
alt_event_type: dialog
Presence-Call-Direction: outbound
answer-state: early
```

### 打开灯:

```xml
sendevent PRESENCE_IN
proto: sip
from: 801@example.com
login: 801@example.com
event_type: presence
alt_event_type: dialog
Presence-Call-Direction: outbound
answer-state: confirmed
```

### 关闭灯:

```xml
sendevent PRESENCE_IN
proto: sip
from: 801@example.com
login: 801@example.com
event_type: presence
Presence-Call-Direction: outbound
alt_event_type: dialog
answer-state: terminated
```

Polycom SIP 3.1之前: SoundPoint IP电话在状态为early和confirmed时会闪烁。

使用SIP 3.1或更高版本: SoundPoint IP电话在状态为early和confirmed时将保持点亮，并显示繁忙图标。
