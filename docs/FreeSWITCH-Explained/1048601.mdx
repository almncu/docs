# 介绍

## 关于

欢迎来到FreeSWITCH社区文档维基，这是一个用于构建从软电话到完整的5级交换机的电话系统构建工具箱。您可以将FreeSWITCH嵌入到软电话（或硬电话）或OpenWRT路由器中，通过FreeTDM和硬件接口（如Sangoma A100系列）连接到PRI电路，或者使用它构建一个办公PBX电话系统 - 从头开始，或使用像FusionPBX这样的封装附加组件。FreeSWITCH拥有庞大而活跃的开发社区，邮件列表，IRC频道，年度大会以及每周视频会议，这些会议都是在FreeSWITCH的实例上托管的。它的授权协议是MPL，即Mozilla公共许可证。

由于FreeSWITCH是一个设计为构建许多不同类型系统的平台引擎，因此它本身就具有模块化的特点。我们的座右铭是：“不要将乐高积木粘在一起。”FreeSWITCH是一个模块化应用程序，其中的模块可以扩展功能，但抽象层可以防止模块间的依赖性。我们的目标是确保一个模块不需要加载另一个模块。

## 什么是FreeSWITCH™？

FreeSWITCH™是一个用C语言从头开始编写的开源通信平台。其核心是根据[MPL 1.1](http://www.opensource.org/licenses/mozilla1.1.php)许可证进行授权，但其他组件有其他许可要求。用户可以在多平台上运行软件，包括Windows、Mac OS X、\*BSD、Debian、Ubuntu和CentOS/Fedora/RHEL。

为了不重复造轮子，我们设计了FreeSWITCH™，以充分利用尽可能多的现有软件库。它具有模块化、可扩展的架构，在其核心中只具备有限和必要的功能。可以使用（或创建）可选模块以添加用户所需的几乎任何功能。

* 默认实现是用于PBX或软交换（实际上是一个B2BUA）  
   * 核心（[libfreeswitch](http://wiki.freeswitch.org/wiki/Libfreeswitch "Libfreeswitch"））可以嵌入到几乎任何可以使用.so或.dll的应用程序中。  
   * 将其转化为软电话、PBX、软交换、会议桥接或自定义电话应用程序。
* 模块系统允许您轻松扩展FreeSWITCH。  
   * 应用程序可以用C/C++、Java、 [.NET](http://wiki.freeswitch.org/wiki/Mod%5Fmanaged "Mod managed")、[Javascript/ECMAScript](http://wiki.freeswitch.org/wiki/Javascript "Javascript")、[Python](http://wiki.freeswitch.org/wiki/Mod%5Fpython "Mod python")、[Perl](http://wiki.freeswitch.org/wiki/Perl "Perl")、Ruby、PHP、[Lua](http://wiki.freeswitch.org/wiki/Lua "Lua")等编写。  
   * 外部系统可以通过TCP [事件套接字](http://wiki.freeswitch.org/wiki/Mod%5Fevent%5Fsocket "Mod event socket")接收事件并/或控制交换机，有很多语言绑定和客户端可用。
* 可以在标准PC上处理成千上万个并发通道，并具有媒体处理能力。
* 与许多不同的产品和协议进行互操作。  
   * 例如[Asterisk](http://www.asterisk.org/)、[Yate](http://yate.null.ro/) 或 [sipXecs](http://www.sipfoundry.org)。  
   * 支持SIP、SCCP、H.323、LDAP、Zeroconf等。  
   * 借助[FreeTDM](http://wiki.freeswitch.org/wiki/FreeTDM "FreeTDM")（一个基于BSD许可的TDM抽象库）或硬件模拟FXO网关，还可以与[PSTN](https://confluence.freeswitch.org/display/FREESWITCH/Glossary#Glossary-pstn)进行接口对接。
* 支持[安全RTP](http://wiki.freeswitch.org/wiki/Secure%5FRTP "Secure RTP") (SRTP)。
* 请参阅完整的[功能列表](http://wiki.freeswitch.org/wiki/Features "Features")获取更多功能。
