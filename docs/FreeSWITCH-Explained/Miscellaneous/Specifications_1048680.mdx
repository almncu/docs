# 规格

## 可能的用途

* 评级和路由服务器
* 转码B2BUA
* IVR和公告服务器
* 会议服务器
* 语音信箱服务器
* 会话边界控制器（SBC）
* 基本的拓扑隐藏会话边界控制器
* 对DAHDI、Khomp、PIKA、Rhino、Sangoma和Xorcom硬件的支持
* 传真服务器
   * T.38网关、终止和发起模式
   * T.30到T.38和T.38到T.30的网关
   * 参见：[mod\_spandsp](https://wiki.freeswitch.org/wiki/Mod%5Fspandsp "Mod spandsp")
* 当然，还可以用作PBX

## 特点

* 支持WebRTC
* 集中式用户/域目录（directory.xml）
* 纳秒级CDR粒度
* 呼叫录音（立体声主叫/被叫左/右声道）
* 高性能多线程核心引擎
* 通过cURL配置您的HTTP服务器（[mod\_xml\_curl](https://wiki.freeswitch.org/wiki/Mod%5Fxml%5Fcurl "Mod xml curl")）
* 使用XML配置文件进行简单解析
* 协议无关
* 可配置的[RFC 2833](https://tools.ietf.org/html/rfc2833)负载类型  
TODO [RFC 2833](https://tools.ietf.org/html/rfc2833)已被[RFC 4733](https://tools.ietf.org/html/rfc4733)取代。
* 发送方音频拨号（DTMF）生成和检测。
* 基于软件的会议（无需硬件要求）
* 宽带语音会议
* 媒体/无媒体模式
* 内置正确的ENUM/ISN拨号功能
* 详细的XML呼叫详单（CDR）
* 半径（Radius）CDR
* 订阅服务器  
   * 共享线路外显（Shared Line Appearances）  
   * 桥接线路外显（Bridged Line Appearances）
* 企业/运营级事件引擎（XML Events，Name Value Events，Multicast Events）
* 可加载的文件格式和流式传输
* 向Shoutcast和Icecast流式传输和播放
* 多语音控制短语接口
* ASR/TTS支持（本地和通过MRCP）
* 基本IP/PBX功能
* 自动接待员
* 自定义振铃音（Early Media）
* 支持XML-RPC
* 支持多种格式的CDR
* SQL引擎提供会话持久性
* 线程隔离
* 并行寻呼
* 串行寻呼
* Mozilla公共许可证
* 支持  
   * 提供付费支持  
   * 免费支持通过IRC和电子邮件
* 多种支持的编解码器  
   * CELT（32 kHz和48 kHz）  
   * G.722.1（宽带）  
   * G.722.1C（宽带32 kHz）  
   * G.722（宽带）  
   * G.711  
   * G.726（16k、24k、32k、48k）AAL2和[RFC 3551](https://tools.ietf.org/html/rfc3551)  
   * G.723.1（透传）  
   * G.729AB（除非使用透传否则需要许可）  
   * AMR（透传）  
   * iLBC  
   * Speex（窄带和宽带）与[RFC 5574](https://tools.ietf.org/html/rfc5574) fmtp支持  
   * LPC-10  
   * DVI4（ADPCM）8 kHz和16 kHz  
   * SILK  
   * OPUS - [RFC 6716](https://tools.ietf.org/html/rfc6716)  
   * 视频编解码器（透传）：  
      * Theora  
         * H.261  
         * H.263  
         * H.264  
         * MP4  
   * 另请参阅：[编解码器](https://wiki.freeswitch.org/wiki/Codecs "Codecs")
* 在两个FreeSWITCH框之间实时迁移呼叫。请参阅[Freeswitch\_HA](https://wiki.freeswitch.org/wiki/Freeswitch%5FHA "Freeswitch HA")。

* 语音信箱  
   * 多租户 - 企业/运营商配置  
   * 时间问候  
   * 紧急消息标记  
   * 电子邮件投递  
   * 在投递之前播放和重新录制消息  
   * 键盘布局可以自定义以适应您的需求  
   * 语音信箱内的回拨支持  
   * 语音信箱的播客 (RSS)  
   * 待接收消息指示器 (MWI)
* 队列支持 (通过[mod\_fifo](https://wiki.freeswitch.org/wiki/Mod%5Ffifo "Mod fifo") 或 [mod\_callcenter](https://wiki.freeswitch.org/wiki/Mod%5Fcallcenter "Mod callcenter"))
* 停车功能 (通过[mod\_fifo](https://wiki.freeswitch.org/wiki/Mod%5Ffifo "Mod fifo"))
* 会议  
   * 基于软件的会议，无需任何硬件需求  
   * 宽带会议  
   * 多个按需或定时会议，带有入场/离场通告  
   * 在会议中播放文件或单个成员  
   * 关系  
   * TTS 集成  
   * 转接  
   * 呼出电话  
   * 可配置的键布局  
   * 每通话的音量、增益和能量水平  
   * 从桥接到会议的过渡  
   * 多方呼出电话  
   * [RFC 4579](https://tools.ietf.org/html/rfc4579) 用于 UAs 的 SIP CC 会议  
   * 自动或按需录音
* RSS 阅读器
* 传真端点，网关和直通模式  
   * T.30 (G.711) 音频传真 (通过[mod\_spandsp](https://wiki.freeswitch.org/wiki/Mod%5Fspandsp "Mod spandsp")), 以前称为 mod\_fax  
   * T.38 传真 (网关，端点和直通模式)

## 协议

* 使用[mod\_sofia](https://wiki.freeswitch.org/wiki/Mod%5Fsofia "Mod sofia")进行SIP通信
   * 支持UDP、TCP、SCTP和TLS传输，以满足SIP的完整标准要求
   * 支持SIP v.2.0（[RFC 3261](https://tools.ietf.org/html/rfc3261)）
   * 支持IPv6
   * 支持SIP会话定时器
   * 支持RTP定时器
   * 支持[RFC 3263](https://tools.ietf.org/html/rfc3263)（SRV和NAPTR）标准
   * 支持[RFC 3325](https://tools.ietf.org/html/rfc3325)标准
   * 支持[RFC 4694](https://tools.ietf.org/html/rfc4694)标准
   * 支持SDES方式的SRTP加密（适用于Polycom、Snom、Linksys和Grandstream等设备）
   * 支持盲注册（Blind SIP Registration）
   * 支持STUN协议（STUN Support）
   * 支持抖动缓冲（Jitter buffer）
   * 支持NAT穿透（NAT Support）
   * 支持分布式SIP注册（Distributed SIP registrations）
   * 支持延迟编解码协商（Late Codec Negotiation）
   * 支持用户账户的多重SIP注册（Multiple SIP registrations per user account）
   * 多租户支持 - 多个SIP用户代理（Multitenancy - Multiple SIP UAs）
   * 支持SIP重新邀请（SIP Reinvites）
   * 可以作为会话边界控制器（SBC）使用
   * 管理实时用户状态（Presence）
   * 支持SIP/SIMPLE（可连接到其他聊天协议）
   * 支持Linksys和Snom的SIP组播呼叫（SIP Multicast Paging）
   * 支持对讲和自动应答（Intercom/AutoAnswer）
   * 支持通话功能，如呼叫保持（Call Hold），盲转移（Blind Transfer），呼叫转发（Call Forward），等等。
* 使用[mod\_dingaling](https://wiki.freeswitch.org/wiki/Mod%5Fdingaling "Mod dingaling")进行Jingle通信
   * 与Google Talk和[Telepathy](http://telepathy.freedesktop.org/)互操作
* 使用[mod\_opal](https://wiki.freeswitch.org/wiki/Mod%5Fopal "Mod opal")进行H.323通信（[opalvoip.org](http://www.opalvoip.org/)）
* 使用[mod\_h323](https://wiki.freeswitch.org/wiki/Mod%5Fh323 "Mod h323")进行H.323通信（[www.h323plus.org](http://www.h323plus.org/)）
* 使用[mod\_opal](https://wiki.freeswitch.org/wiki/Mod%5Fopal "Mod opal")进行IAX2通信（[opalvoip.org](http://www.opalvoip.org/)）
* [mod\_skinny](https://wiki.freeswitch.org/wiki/Mod%5Fskinny "Mod skinny") \- 用于Skinny呼叫控制协议（SCCP）

## 跨平台

* 在MSVC上原生构建Windows
* 在macOS、Linux、Solaris和\*BSD上构建。

## 最低/推荐系统要求

* 32位操作系统（建议使用64位）
* 512MB内存（建议使用1GB）
* 50MB磁盘空间

系统要求取决于您的部署需求。我们建议您计划50%的工作循环。

## 性能

* 在超过100小时的负载测试下进行
* 超过10,000,000次调用
* 超过50 CPS的频率

性能会根据应用程序而有所不同。您需要根据特定情况进行测试。
