## 终端点、拨号计划、会话

摘自《FreeSWITCH 1.0.6》一书的第10页：

"终端点模块是至关重要的，它们增加了一些关键功能，使FreeSWITCH成为如今强大的平台。这些模块的主要作用是将某些常见的通信技术规范化为一种我们称之为会话的共同抽象实体。会话代表了FreeSWITCH与特定协议之间的连接。FreeSWITCH附带了几个终端点模块，实现了诸如SIP、H.323、Jingle（谷歌通信）等多种协议。我们将花一些时间来研究其中一个较受欢迎的模块，名为mod\_sofia。

"Sofia-SIP（[http://sofia-sip.sourceforge.net](http://sofia-sip.sourceforge.net/)）是由诺基亚赞助的开源项目，旨在为Session Initiation Protocol（SIP）创建一个编程接口。在FreeSWITCH中，我们使用这个库在一个我们称为mod\_sofia的模块中。该模块注册到FreeSWITCH中必要的所有挂接点，并将原生的FreeSWITCH结构转换成SIP结构，反之亦然。配置信息是从中央的FreeSWITCH配置文件中提取的，这允许mod\_sofia加载用户定义的首选项和连接详情。这使得FreeSWITCH能够接受SIP电话和设备的注册，注册到其他SIP终端点（如服务提供商），发送通知，并为电话提供诸如语音邮件等服务。

当FreeSWITCH与另一个SIP设备之间建立SIP呼叫时，它将在FreeSWITCH中显示为活动会话。如果呼叫是呼入的，它可以被转接或桥接到交互式语音响应（IVR）菜单、保持音乐或一个或多个分机，尽管还有很多其他选项可用。让我们看一个典型的场景，在这个场景中，一个注册为分机2000的SIP电话拨打分机2001，希望建立一个呼叫。
