## 规划新的FreeSWITCH配置

### 规划您的拨号计划

这可能因为是用于商业还是私人使用，以及所在国家的不同而大为不同，不同地方期望不同的拨号计划。因此，这个问题真的值得您首先考虑。您需要配置一些分机，并决定您想如何拨打电话，是否打出外线需要使用一个数字，或者您希望直接拨打普通号码而无需透明。在后一种情况下，您还需要使分机和其他“服务”号码适应，以免干扰普通电话号码。

### 确定您将配置哪些网关来拨打电话

很可能你将需要配置一个或多个网关来进行外拨。你可能打算使用一个主网关和一个备用网关，或者为特定目的地使用特定的网关。有很多可能性。然而，你应该确定你想要做什么。

### 确定人们如何拨打FreeSWITCH服务器

你可能有一个或多个接口，每个接口都对谁可以使用它有不同的“限制”。来自网关的呼入呼叫可能只能从已知来源于该网关的地址中收到。你可能希望一些分机可以从外部访问，你可能只希望某些分机可以访问，或者你可能希望有一个全局的入口号码。

### 确定你想要实现的额外功能

这涉及到语音信箱、电话会议、IVR系统等等，以及允许谁使用它们。

### 安全性

不同的人可能对需要应用的安全措施有不同的想法。然而，考虑一下你的系统如何被访问以及你可能希望采取什么保护措施是值得的。其中一些可能是通过FreeSWITCH来实现的，其中一些可能是通过你的操作系统或外部防火墙来实现的。

* 你是否希望有一个防火墙。某种类型的防火墙可能是个不错的主意。然而，如果你在NAT连接的背后，请不要认为自己完全安全免受攻击。出站的UDP流量通常可以打开回到你系统的入站连接。
* 限制来自分机的入口注册和/或呼叫的速率
* 限制来自入站供应商的呼入呼叫速率
* 限制特定分机的出站呼叫速率
* 限制通过供应商的并发出站呼叫速率

### 初始配置

目前还没有一个系统能够根据不同的配置需求来实现“空配置”。这样做可能会很不错，所以很可能您需要从默认配置开始，并根据您上面概述的需求进行调整。此外，还希望有一些测试工具，以便您可以查看特定行为是否按预期工作，同时也排除了非预期行为。前者比较容易检查，而后者可能会更难实现。
